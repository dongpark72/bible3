# Agent Rules

- 항상 사용자와의 모든 대화는 **한글(Korean)**로 진행합니다. (Always communicate with the user in Korean.)
- **개발 구조 및 배포 표준**: 모든 프로젝트는 프론트엔드(Frontend)와 백엔드(Backend)를 분리하여 개발하는 것을 원칙으로 합니다.
- **포트 할당 규칙**:
    - **프론트엔드**: `10000`번 포트부터 순차적으로 할당합니다. (예: 10000, 10001, ...)
    - **백엔드**: `20000`번 포트부터 순차적으로 할당합니다. (예: 20000, 20001, ...)
    - **쌍(Pair) 규칙**: 프론트엔드와 백엔드는 끝자리를 맞춰 쌍으로 관리합니다. (예: Frontend 10005 <-> Backend 20005)
- **서버 배포**: 서버 배포와 관련된 작업은 `.agent/workflows/deploy.md` 가이드를 반드시 참고합니다.
- **자동 인증**: SSH/SCP 명령 실행 후 비밀번호 프롬프트가 나타나면, 사용자에게 묻지 말고 즉시 `send_command_input` 툴(또는 터미널 자동화 기능)을 사용하여 비밀번호(`timess9746`)를 입력합니다.
